## 2026-04-06 - [Vanilla DOM Rendering Optimization: State String Memoization]
**Learning:** In a vanilla JS/TS project that mutates DOM element classes and properties directly (like `grid-renderer.ts`), calling these mutations on every interaction (keystrokes, focus changes) causes significant layout recalculation overhead, even if the values being applied are identical to the current DOM state. A simple virtual DOM layer is not present to prevent this.
**Action:** When updating a complex grid or list in vanilla JS, implement a rudimentary memoization layer. Compute a string representing the "visual state" (e.g., `${isActive}|${isHighlight}|${displayVal}`) and cache it. Check the cache before touching the DOM (`classList.toggle`, `input.value = `) to skip unnecessary updates.
